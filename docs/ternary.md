## 三目运算符（?:）书写规范与限制

本项目为了提升可读性，对三目运算符进行了额外限制：

- **三目运算符必须独立写在一行或严格的多行格式中**
- **禁止把三目运算符当作函数/方法/构造调用的参数（禁止 PHP 式写法）**

---

## 1) 允许的写法：单行

单行时要求 `?` 与 `:` 都在同一行：

```longlang
value := (x > y) ? true : false
```

---

## 2) 允许的写法：多行（`?` 与 `:` 都必须提行）

多行时要求 `?` 和 `:` **各自单独换行**（`?` 提行、`:` 也提行）：

```longlang
value := (x > y)
    ? true
    : false
```

---

## 3) 禁止的写法：混写（`?` 提行但 `:` 不提行）

下面这种写法会被判定为不可读，因此禁止：

```longlang
value := (x > y)
    ? true : false
```

---

## 4) 禁止的写法：把三目运算符当作函数/方法参数

禁止把三目运算符当作参数传入函数/方法/构造：

```longlang
get_xxx(age > 20 ? true : false)
```

如果需要，请先把三目表达式写成一个独立赋值，再把变量传入函数：

```longlang
flag := (age > 20) ? true : false
get_xxx(flag)
```
